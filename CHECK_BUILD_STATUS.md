# 🔍 检查GitHub Actions构建状态和下载APK

## 📋 快速操作指南

### 第1步：访问GitHub仓库
打开浏览器访问：
```
https://github.com/deanGuo1987/exercise-tracker
```

### 第2步：查看Actions状态
1. 点击仓库页面顶部的 **"Actions"** 标签页
2. 你会看到工作流列表，找到 **"Build Android APK"**
3. 查看状态图标：
   - 🟡 **黄色圆圈** = 正在运行中
   - ✅ **绿色对勾** = 构建成功
   - ❌ **红色叉号** = 构建失败

### 第3步：下载APK文件

#### 如果构建成功（绿色✅）：
1. 点击成功的构建任务
2. 滚动到页面底部
3. 找到 **"Artifacts"** 部分
4. 点击下载以下文件：
   - `exercise-tracker-debug-apk` - 调试版本（推荐）
   - `exercise-tracker-release-apk` - 正式版本

#### 如果构建失败（红色❌）：
1. 点击失败的构建任务
2. 查看错误日志
3. 通常重新运行即可解决

## 🕐 预计时间

- **构建时间**: 3-8分钟
- **下载时间**: 1-2分钟（取决于网速）

## 📱 APK文件信息

下载的文件是zip格式，需要解压：

### Debug APK (推荐测试使用)
- **文件名**: `app-debug.apk`
- **大小**: 约8-15MB
- **用途**: 开发测试版本，包含调试信息

### Release APK (正式版本)
- **文件名**: `app-release-unsigned.apk`
- **大小**: 约6-12MB
- **用途**: 正式版本，体积更小

## 🔧 如果遇到问题

### 问题1: 构建时间过长（超过10分钟）
**解决方案**: 
- 刷新页面查看最新状态
- 可能是GitHub服务器繁忙，耐心等待

### 问题2: 构建失败
**解决方案**:
1. 点击失败的构建查看日志
2. 在Actions页面点击 "Re-run jobs"
3. 如果持续失败，检查错误信息

### 问题3: 找不到Artifacts
**解决方案**:
- 确保构建显示绿色✅
- 滚动到构建详情页面最底部
- Artifacts部分在页面底部

### 问题4: 下载的是zip文件
**解决方案**:
- 这是正常的，GitHub会自动打包
- 解压zip文件获得APK
- APK文件可以直接安装到Android设备

## 📲 安装APK到手机

### 准备工作
1. **启用开发者选项**:
   - 设置 → 关于手机 → 连续点击"版本号"7次

2. **允许安装未知来源**:
   - 设置 → 安全 → 未知来源（或"安装未知应用"）

### 安装步骤
1. 将APK文件传输到Android设备
2. 在设备上打开文件管理器
3. 找到APK文件并点击
4. 按照提示完成安装
5. 安装后授予通知权限

## 🎯 应用功能验证

安装后测试以下功能：
- ✅ 应用启动正常
- ✅ 日历界面显示
- ✅ 点击日期弹出运动选择
- ✅ 记录运动时长（20/30/40分钟）
- ✅ 数据保存和显示
- ✅ 通知权限设置

## 🚀 自动发布版本（可选）

如果想创建正式发布版本：

```bash
# 在本地项目目录运行
git tag v1.0.0
git push origin v1.0.0
```

这将在GitHub Releases页面创建正式版本，更容易下载和分享。
